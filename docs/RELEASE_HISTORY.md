## Sammanfattning

Vardagsstyrka är ett tidigt app-projekt. Fokus är att etablera ett tydligt kärnflöde, enkel användning, mobilvänlig upplevelse och stabil dokumentation från början.

## Historik

### Tidig grund

Typ: Projektstart

- Repository skapat.
- Grundläggande kodbas etablerad.
- Backlog påbörjad.
- Iterativ utveckling enligt prioriterade batcher.
- PROJECT_INFO och RELEASE_HISTORY har skapats för att minska framtida historikskuld.

### Kommande historik

Typ: Fortsatt utveckling

- Större funktioner dokumenteras här när de implementeras.
- Versionshöjningar dokumenteras här.
- Viktiga tekniska beslut dokumenteras här.

## Releaseprinciper

1. Patchversion används för mindre fixar.
2. Minorversion används för sammanhängande batcher eller märkbara förbättringar.
3. Majorversion används för större förändringar i struktur eller användarupplevelse.
4. Synlig version i appen ska hållas i takt med kodversion när sådan finns.
5. PROJECT_INFO ska uppdateras när releasehistoriken ändrar riktning.

## Kommande förbättringar att dokumentera

- Första stabila kärnflödet.
- Startsida och primär användarresa.
- Mobil-UX.
- Historik eller progression om det införs.
- Teknisk skuld och strukturförbättringar.
